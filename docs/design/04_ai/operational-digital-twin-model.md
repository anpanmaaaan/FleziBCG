## Twin starting point

The digital twin should start as an operational twin, not a 3D-first system.

## Core twin objects
- line / station / machine / resource
- execution run / step / operation context
- WIP / queue / block / delay state
- material lot/batch context later

## Design rule

Twin modeling must support both discrete and process/batch plants over time.
