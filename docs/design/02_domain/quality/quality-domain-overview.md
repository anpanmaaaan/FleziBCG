Status: Canonical entry document for the quality domain.

## 1. Purpose

This document explains:
- what quality owns
- what Quality Lite covers now
- how quality interacts with execution
- how quality remains compatible with both discrete and process-oriented plants

## 2. Domain purpose

The quality domain owns:
- QC applicability
- measurement/inspection input
- backend evaluation of pass/fail/hold
- disposition flow
- accepted-good implications

## 3. Interaction with execution

Quality is execution-adjacent and policy-driven.
Quality must not overwrite execution state names directly.
Quality interacts through orthogonal dimensions, gating, and accepted-good derivation.

## 4. Platform-neutral rule

Quality must remain compatible with:
- operation-level discrete checks
- batch/lot-oriented process checks
- held/pending quantity derivation later
